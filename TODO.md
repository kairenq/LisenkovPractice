# IS: Учет расходуемого материала (React + Electron)

## 0) Архитектура и сборка
- [ ] Создать монорепо: `apps/desktop` (Electron) + `packages/shared`
- [ ] Electron: main process + preload (contextBridge), renderer = React
- [ ] Хранилище данных: SQLite (локально) + миграции (рекомендовано для практики)
- [ ] GitHub Actions: build Windows `.exe` (electron-builder), upload artifact + release draft

## 1) Роли и доступ
- [ ] Роли: `кладовщик`, `бухгалтер`, `руководитель`, `администратор`
- [ ] RBAC: матрица прав
  - админ: управление пользователями + всё
  - кладовщик: материалы/приход/расход/остатки
  - бухгалтер: отчеты + просмотр операций
  - руководитель: отчеты + остатки

## 2) Авторизация (обязательная по UML)
- [ ] Экран входа (логин/пароль)
- [ ] Хеширование паролей (bcrypt)
- [ ] Сессия (token в памяти/secure storage), кнопка “Выйти”
- [ ] Экран “Нет доступа” при запрете по роли

## 3) Справочник материалов (обязательный)
- [ ] CRUD материалов: добавить/редактировать/удалить
- [ ] Поля: наименование, единица измерения, категория (опц), минимальный остаток (опц)
- [ ] Поиск/фильтр/сортировка
- [ ] Валидации (пустые/отрицательные значения)

## 4) Учет прихода материалов (обязательный)
- [ ] Форма “Приход”: материал, количество, дата, комментарий/поставщик (опц)
- [ ] Запись операции прихода
- [ ] Автообновление остатков

## 5) Учет расхода материалов (обязательный)
- [ ] Форма “Расход/Списание”: материал, количество, дата, причина/подразделение (опц)
- [ ] Запрет списания больше остатка (или предупреждение + запрет)
- [ ] Запись операции расхода
- [ ] Автообновление остатков

## 6) Просмотр остатков (обязательный)
- [ ] Таблица остатков по всем материалам
- [ ] Подсветка материалов ниже минимального остатка (если поле есть)
- [ ] Фильтры: по категории/по складу (если добавим склад), по низким остаткам

## 7) Формирование отчетов (обязательный)
- [ ] Отчет о расходе материалов за период (дата с/по, материал/категория)
- [ ] Отчет по остаткам на дату
- [ ] Экспорт: PDF + CSV (для “печати отчетов” в практике)
- [ ] Шаблон отчета: шапка, период, таблица, итоги

## 8) Управление пользователями (обязательный по UML)
- [ ] CRUD пользователей: логин, роль, активен/не активен
- [ ] Сброс пароля админом
- [ ] Аудит: кто создал/изменил пользователя (опц, но круто для практики)

## 9) Системные штуки для отчета по практике
- [ ] Логи действий (минимум): вход, добавление материала, приход, расход, отчет
- [ ] Обработчик ошибок + окно “Ошибка подключения/ошибка БД” (как у тебя в Рис. 6.6)
- [ ] Набор тестовых данных (seed) для демонстрации

## 10) Скрины под отчет (что обязательно должно быть в UI)
- [ ] Окно авторизации (для Рисунок 4.1 / 6.1)
- [ ] Главное окно (Рисунок 6.2)
- [ ] Добавление материала (под твой Рисунок 6.3, но подпись исправим)
- [ ] Результат добавления (Рисунок 6.4)
- [ ] Сообщение об ошибке (пустые поля / неверный логин) (Рисунок 6.5)
- [ ] Ошибка БД/подключения (Рисунок 6.6)

## 11) CI/CD (GitHub Actions)
- [ ] workflow: install → lint → test → build renderer → build electron → package `.exe`
- [ ] cache npm
- [ ] artifact: `.exe` + portable zip (опц)
